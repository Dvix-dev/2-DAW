AF 17.1

Crear un sitio web para recetas de cocina.

Se puede emplear una plantilla de bootstrap para la interfaz o diseñar dicha interfaz

desde cero, como se quiera. Igual para los formularios a utilizar.

Para almacenar la información de las recetas vamos a usar una base de datos. La base

de datos se llamará “cocina” y dentro tendremos una serie de tablas que se detallan

a continuación:

“Se desea tener información sobre las diferentes recetas que queremos almacenar. De

cada receta se almacenará: Cod_receta (clave primaria), nombre, descripción, foto,

documento_pdf, tiempo.

Se almacenará información sobre las diferentes categorías en las que podemos

clasificar las recetas. De cada categoría se desea almacenar: Cod_categoria, nombre,

descripción.

Una receta puede ser de varias categorías. De una misma categoría podríamos tener

varias recetas.”

Cod_receta: Será un número. Del 1 en adelante.

Nombre de la receta: Cadena de caracteres variable.

Descripción de la receta: Cadena de caracteres variable.

Foto: Nombre y extensión de la foto de la receta. Extensiones permitidas: (.jpg, .jpeg,

.png, .gif).

Documento_pdf: Nombre y extensión del documento que muestra de manera extensa

la receta. Extensiones permitidas: (.pdf).

Tiempo: Número del 1 en adelante.

Cod_categoria: Será un número. Del 1 en adelante.

Nombre Categoría: Cadena de caracteres variable. (corto). Las categorías serán:

(Verduras, Arroces, Guisos, Postres, Bebidas).

Una misma receta puede pertenecer a varias categorías, por

ejemplo, un estofado de ternera y verduras podría tener la

categoría: verduras, guisos

Descripción de la Categoria: Cadena de caracteres variable.

dicho cambio, así como efectuar el cambio también en la base de datos de forma

adecuada.

Hacer igual para las categorías, pulsando en el botón “Operaciones sobre Categorías”.
